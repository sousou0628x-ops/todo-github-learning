GitHub学習プロジェクト - 教育計画
=====================================

プロジェクト概要：
シンプルなToDoリストアプリケーション（HTML/CSS/JavaScript）を通じて、
GitHubの基本機能を段階的に学習する

=====================================
ステップ1: 基本的なGitワークフロー
=====================================

学習目標：
- Gitの基本コマンドの理解
- ブランチの作成と切り替え
- コミットの作成
- マージの実践
- コンフリクトの解決

実践内容：
1. ローカルリポジトリの初期化
   - git init
   - 初回コミット

2. ブランチ戦略の理解
   - main（本番）ブランチ
   - develop（開発）ブランチ
   - feature/機能名（機能開発）ブランチ

3. 実践演習：
   - feature/add-todo-form ブランチでフォーム機能を追加
   - feature/add-styling ブランチでスタイリングを追加
   - developブランチにマージ
   - mainブランチにマージ

4. GitHubへのプッシュ
   - リモートリポジトリの設定
   - git push の実践

成果物：
- 基本的なToDoリストアプリ（HTML構造のみ）
- .gitignore ファイル
- README.md（プロジェクト説明）

=====================================
ステップ2: GitHub Actions（CI/CD）
=====================================

学習目標：
- GitHub Actionsの基本概念
- ワークフローファイルの作成
- 自動テストの設定
- 自動デプロイの理解

実践内容：
1. GitHub Actionsの基礎
   - .github/workflows/ ディレクトリ構造
   - YAMLファイルの書き方
   - トリガー（push, pull_request等）

2. 実践演習：
   - HTMLバリデーションの自動実行
   - JavaScriptのLint（ESLint）
   - GitHub Pagesへの自動デプロイ

3. ワークフロー例：
   - コードプッシュ時の自動チェック
   - Pull Request作成時のテスト実行
   - mainブランチマージ時のデプロイ

成果物：
- .github/workflows/ci.yml（テスト自動化）
- .github/workflows/deploy.yml（デプロイ自動化）
- package.json（依存関係管理）

=====================================
ステップ3: Issue/Pull Requestテンプレート
=====================================

学習目標：
- Issueの作成と管理
- Pull Requestのワークフロー
- コードレビューの基礎
- プロジェクト管理の実践

実践内容：
1. Issueテンプレートの作成
   - バグレポート用テンプレート
   - 機能リクエスト用テンプレート
   - ラベルの活用

2. Pull Requestテンプレート
   - 変更内容の説明フォーマット
   - チェックリスト
   - レビュアーの指定

3. 実践演習：
   - Issueを作成（例：「削除機能の追加」）
   - Issueに紐づくブランチを作成
   - Pull Requestを作成
   - セルフレビュー
   - マージ

4. プロジェクトボードの活用
   - To Do / In Progress / Done
   - Issueの進捗管理

成果物：
- .github/ISSUE_TEMPLATE/bug_report.md
- .github/ISSUE_TEMPLATE/feature_request.md
- .github/pull_request_template.md
- 実際のIssueとPull Requestの履歴

=====================================
ステップ4: README、ライセンス、.gitignore
=====================================

学習目標：
- プロフェッショナルなREADMEの作成
- オープンソースライセンスの理解
- .gitignoreの適切な設定
- ドキュメンテーションのベストプラクティス

実践内容：
1. README.mdの充実
   - プロジェクト概要
   - インストール方法
   - 使用方法
   - スクリーンショット
   - 貢献ガイドライン
   - バッジの追加（ビルドステータス等）

2. ライセンスの選択
   - MITライセンス（推奨）
   - LICENSE ファイルの作成

3. .gitignoreの最適化
   - node_modules/
   - .DS_Store（Mac）
   - エディタ設定ファイル
   - 環境変数ファイル

4. 追加ドキュメント
   - CONTRIBUTING.md（貢献方法）
   - CODE_OF_CONDUCT.md（行動規範）
   - CHANGELOG.md（変更履歴）

5. GitHub機能の活用
   - About セクションの設定
   - Topics（タグ）の追加
   - GitHub Pagesの有効化

成果物：
- 完全なREADME.md
- LICENSE ファイル
- .gitignore ファイル
- CONTRIBUTING.md
- プロジェクトの公開準備完了

=====================================
最終成果物
=====================================

完成するもの：
1. 動作するToDoリストアプリケーション
   - タスクの追加
   - タスクの完了/未完了切り替え
   - タスクの削除
   - ローカルストレージでの永続化

2. 完全なGitHub環境
   - 適切なブランチ戦略
   - 自動化されたCI/CD
   - Issue/PR管理体制
   - プロフェッショナルなドキュメント

3. 学習の証跡
   - コミット履歴
   - ブランチ履歴
   - Issue/PR履歴
   - GitHub Actionsの実行履歴

=====================================
学習のポイント
=====================================

- 各ステップで実際に手を動かすこと
- コミットメッセージは明確に書く
- ブランチ名は目的がわかるように命名
- Pull Requestには詳細な説明を記載
- GitHub Actionsのログを確認する習慣
- ドキュメントは常に最新に保つ
