# GitHub実践カリキュラム - 応用編

基本の4ステップを完了したあなたへ、さらに実践的なスキルを学びます。

---

## 📋 完了した基本カリキュラム

✅ **ステップ1: 基本的なGitワークフロー**
- コミット履歴で変更を追跡
- いつでも過去に戻れる安心感

✅ **ステップ2: ブランチ戦略**
- 安全に実験できる並行世界
- 失敗を恐れない開発

✅ **ステップ3: Issue/Pull Request**
- 変更の理由を記録
- 未来の自分が理解できる

✅ **ステップ4: GitHub Actions（自動デプロイ）**
- プッシュするだけで自動デプロイ
- 手動作業からの解放

---

## 🚀 実践カリキュラム

### ステップ5: コンフリクト解決（マージの衝突）

**失敗シナリオ：**
「2つのブランチで同じファイルを変更したら、マージできない！」

**学ぶこと：**
- コンフリクトが起こる仕組み
- コンフリクトの解決方法
- コンフリクトを避ける開発方法

**実践内容：**
1. わざとコンフリクトを起こす
2. コンフリクトマーカーを理解する
3. 手動で解決する
4. マージを完了する

---

### ステップ6: Git履歴の操作（過去を変える）

**失敗シナリオ：**
「間違えてパスワードをコミットしてしまった！」
「コミットメッセージを間違えた！」

**学ぶこと：**
- `git commit --amend`（最後のコミットを修正）
- `git reset`（コミットを取り消す）
- `git revert`（安全に取り消す）
- `git rebase`（履歴を綺麗にする）

**実践内容：**
1. 間違ったコミットを修正
2. 複数のコミットをまとめる
3. コミット順序を変更する

---

### ステップ7: チーム開発シミュレーション

**失敗シナリオ：**
「他の人の変更と自分の変更が衝突した！」
「誰がこのバグを入れたの？」

**学ぶこと：**
- `git pull`（他の人の変更を取得）
- `git blame`（誰がいつ変更したか）
- `git stash`（作業を一時保存）
- コードレビューの実践

**実践内容：**
1. 複数ブランチで並行開発
2. Pull Requestでレビュー
3. コンフリクトを解決してマージ

---

### ステップ8: GitHub高度な機能

**学ぶこと：**
- GitHub Projects（プロジェクト管理）
- GitHub Discussions（議論の場）
- GitHub Wiki（ドキュメント）
- GitHub Releases（バージョン管理）
- GitHub Sponsors（支援）

**実践内容：**
1. プロジェクトボードでタスク管理
2. Wikiでドキュメント作成
3. Releaseでバージョン公開

---

### ステップ9: オープンソースへの貢献

**学ぶこと：**
- Fork（他人のリポジトリをコピー）
- Pull Requestを他のプロジェクトに送る
- オープンソースのマナー
- コントリビューターになる

**実践内容：**
1. 有名なOSSプロジェクトを探す
2. Issueを読む
3. 小さなバグ修正やドキュメント改善
4. Pull Requestを送る

---

### ステップ10: ポートフォリオ作成

**学ぶこと：**
- プロフィールREADME
- ピン留めリポジトリ
- GitHub Statsの表示
- 就職・転職でアピール

**実践内容：**
1. プロフィールを充実させる
2. 代表的なプロジェクトをピン留め
3. READMEを魅力的に書く
4. GitHub Actionsでバッジを表示

## 🎓 学習のヒント

1. **焦らない**
   - 一度に全部やる必要はない
   - 必要になったら学ぶ

2. **実践する**
   - 読むだけでなく、実際に手を動かす
   - 失敗してもGitがあれば戻せる

3. **記録する**
   - 学んだことをメモ
   - 自分用のWikiを充実させる

4. **楽しむ**
   - GitHubは便利なツール
   - 使いこなせると開発が楽しくなる
