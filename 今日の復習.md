# 今日の学習復習

## ✅ 学んだこと

### 1. Gitの基本コマンド

```bash
# 状態確認（最重要！）
git status

# 変更内容を確認
git diff

# ステージング（コミット準備）
git add ファイル名
git add .  # 全ての変更

# コミット（記録）
git commit -m "メッセージ"

# GitHubにプッシュ
git push origin ブランチ名

# GitHubから取得
git pull origin main
```

### 2. ブランチの使い方

```bash
# 新しいブランチを作成して切り替え
git checkout -b feature/新機能名

# ブランチ一覧
git branch

# ブランチ切り替え
git checkout main

# マージ
git merge feature/新機能名

# ブランチ削除
git branch -d feature/新機能名  # ローカル
git push origin --delete feature/新機能名  # リモート
```

### 3. Issue駆動開発

```
1. GitHubでIssue作成（#1）
   ↓
2. ブランチ作成
   git checkout -b feature/1-説明
   ↓
3. コードを書く
   ↓
4. コミット（Issue番号を含める）
   git commit -m "Fix #1: 説明"
   ↓
5. プッシュ
   git push origin feature/1-説明
   ↓
6. Pull Request作成
   本文に "Closes #1"
   ↓
7. マージ → Issueが自動で閉じる
```

### 4. GitHub Actions（自動デプロイ）

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      - uses: actions/deploy-pages@v4
```

**結果：** `git push`するだけで自動デプロイ！

## 🎯 重要なポイント

### ローカルとリモート

```
ローカル（あなたのPC）    リモート（GitHub）
┌─────────────┐          ┌─────────────┐
│ main        │ ─push→   │ main        │
│             │ ←pull─   │             │
└─────────────┘          └─────────────┘
```

### IssueとPull Requestの番号

- Issue #1 → Pull Request #2 → Issue #3...
- 同じ番号空間を共有
- どちらも「会話スレッド」

### コミットメッセージの魔法

```bash
# Issueにリンク
git commit -m "タスク追加 #5"

# Issueを自動で閉じる
git commit -m "Fix #5: バグ修正"
git commit -m "Closes #5: 機能実装"
```

## 💡 覚えておくべきコマンド Top 10

1. `git status` - 現在の状態確認
2. `git add .` - 全ての変更をステージング
3. `git commit -m "メッセージ"` - コミット
4. `git push origin main` - GitHubにプッシュ
5. `git pull origin main` - 最新版を取得
6. `git checkout -b ブランチ名` - 新ブランチ作成
7. `git checkout main` - mainに戻る
8. `git merge ブランチ名` - マージ
9. `git log --oneline` - 履歴確認
10. `git restore ファイル名` - 変更を取り消す
